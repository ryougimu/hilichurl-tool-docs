# 第7章：設定リファレンス

## プロジェクトディレクトリ構造

プロジェクト作成後、プロジェクトディレクトリのファイル構造は次のようになります。

```
{プロジェクト名}/
├── project.json                 プロジェクト設定ファイル
├── characters/                  キャラクターデータディレクトリ
│   └── {キャラクターID}/
│       ├── data.json            キャラクターデータ（属性/戦闘設定/登場方式など）
│       ├── event.json           イベントメタデータ（有効状態/追加項目）
│       ├── Event/               イベントブロックファイル（イベントごとに1つのJSON）
│       ├── Unit/                戦闘アイコン（小）
│       ├── Unit_H/              戦闘アイコン（大）
│       ├── Figure/              立ち絵
│       ├── CG/                  イベントCG画像
│       ├── Mating/              交配シーン画像
│       ├── Animated/            サムネイル（自動生成）
│       ├── Icon/                アバターアイコン
│       ├── Torture/             拷問画像
│       └── Voice/               キャラクターボイス
├── event_assets/                プロジェクトレベル共有イベントリソース
│   ├── Background/              共有背景
│   ├── Figure/                  共有立ち絵
│   └── Voice/                   共有ボイス
└── export/                      エクスポート出力ディレクトリ（各エクスポート前にクリア）
```

## イベント

現在のイベントカテゴリーとサブイベントはプリセットです。カスタムは作成できません。詳細なイベントリストは[イベントエディタ](04_h5_event_editor.md)を参照してください。

イベントファイルは `characters/{キャラクターID}/Event/` に保存され、ファイル名は `{イベントID}_{キャラクターID}.json` 形式です（例：`captured_mika.json`）。

## ブロックデータ形式

イベントファイルでは、ブロックは JSON 配列として保存され、各要素はテキストコマンドの1行です。

```json
{
  "event": [
    "background:bg_forest.webp",
    "name:Alice",
    "dialog:こんにちは！",
    "",
    "CG:mika_01_1.webp",
    "sound:sound=slash_1 vol=0",
    "dialog:何が起こった？",
    ""
  ]
}
```

- 各配列要素は1つのブロックに対応  
- 空文字列 `""` はクリック待ち（ページ区切り）  
- 形式は `コマンド:パラメータ`、複数パラメータはスペース区切りのキーバリューペア  

## キャラクターデータ構造

キャラクターデータは `characters/{キャラクターID}/data.json` に保存されます。主なトップレベルフィールド：

| フィールド | 説明 |
|-----------|------|
| translation | キャラクター名翻訳（chs/en/jp） |
| heroine_info | キャラクター属性（星ランク/リーダーシップ/元素など） |
| unit_info | 戦闘設定（攻撃方式/スキル/装備） |
| unit_info_h | 戦闘設定（敵視点） |
| appearance | 登場設定（登場条件/位置/ミニオン/速度） |
| brothel_info | アイテムドロップ確率リスト |
