# 第五章：导出与打包

> **警告：每次导出会完全清空 `export/` 目录！** 如果你在导出目录中做过手动修改，请先将文件复制到其他位置，否则会永久丢失。

角色数据编辑完成后，需要导出为游戏可加载的格式。点击主界面顶部的「Pack」按钮打开导出对话框。

## 导出设置

![项目导出框](images/05/1.png)

| 设置项 | 说明 |
|--------|------|
| 包 ID（Package ID） | 导出包的标识符（如 `21`），对应输出文件名 `patch_21.pck` |
| 打包为 PCK | 是否将资源打包为单个 PCK 文件。关闭则输出散装资源文件 |
| 简单打包（Simple Pack） | 仅在开启 PCK 时可用，生成体积更小的 PCK（详见下方说明） |

输出路径固定为项目目录下的 `export/` 文件夹。

对话框会记住上次的设置，下次打开时自动恢复。点击「查看导出说明」按钮可查看详细的导出指南。

## 导出流程

导出分为以下阶段自动执行（**注意：首先会清空 `export/` 目录**）：

1. **资源整合** —— 扫描所有角色目录，将资源文件复制到输出目录，生成 Install 脚本
2. **项目配置生成** —— 生成 Godot 项目配置文件（PCK 打包所需）
3. **PCK 打包** —— 调用 Godot 引擎将资源打包为 `.pck` 文件（仅开启 PCK 时）
4. **事件文件输出** —— 将所有已启用的事件导出为独立的 JSON 文件

## 打包模式

### PCK 模式（默认）

所有资源打包为单个 `patch_{id}.pck` 文件，同时输出 Install 脚本和事件文件。

### 简单打包模式

生成体积更小的 PCK 文件，仅包含脚本和语音资源。图片等较大的资源文件单独输出，不打入 PCK。适合需要减小 PCK 体积的场景。

### 非 PCK 模式

不打包，直接输出散装的资源文件夹 + Install 脚本 + 事件文件。

## 输出内容

导出完成后，`export/` 目录下包含：

| 内容 | 说明 |
|------|------|
| `patch_{id}.pck` | 打包后的资源文件（仅 PCK 模式） |
| `Install_{id}.gd` | 安装脚本，游戏加载时执行，注册角色数据 |
| `Event/*.json` | 已启用的事件文件，每个事件一个文件 |
| 资源文件夹 | Unit/Figure/CG/Voice 等资源目录 |

## 前置要求

- 导出 PCK 需要配置 Godot 可执行文件路径（在设置中配置）
- 至少有一个角色且数据已保存
