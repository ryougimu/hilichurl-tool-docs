# 第三章：基础数据来源（Options）

> **注意**：Options 的目录结构仍在开发中，后续版本可能会调整。你不需要深入理解其内部细节，只需知道它的作用和如何配置即可。

## 什么是 Options？

编辑器本身只是一个工具，它需要知道游戏里有哪些英雄模板、物品、音效、背景图等基础信息，才能在编辑界面中提供对应的选项供你选择。

这些基础信息统一存放在一个叫 **options** 的目录中。你可以把它理解为编辑器的"素材库"——编辑器从中读取数据来填充下拉列表、加载图标、播放音效预览等。

## 如何获取

下载 options 配置包，解压到本地任意位置，然后在编辑器的设置中指定该目录路径即可。具体下载地址见[附录 - 下载](08_appendix.md#downloads)。

## 目录结构

options 目录包含以下几个区域：

```
options/
├── generated/       英雄、物品、技能等游戏基础数据
├── audio/           音效和 BGM 资源及选项
├── images/          背景图等图片资源及选项
├── event/           事件模板文件
├── template/        新建角色时使用的默认模板
└── md/              编辑器内置的使用说明文档
```

### generated/ —— 游戏基础数据

最核心的部分，包含英雄模板、物品、技能、随从单位等游戏基础数据及其图标资源。新建角色时选择的英雄模板、战斗配置中的各种选项、物品掉落列表等都来自这里。

### audio/ —— 音效与 BGM

事件编辑器中可供选择的音效和 BGM 资源。

### event/ —— 事件模板

预定义的事件模板文件，每个文件对应一种事件类型（如 `captured.json`、`give_birth_a.json` 等）。新建事件时可基于这些模板快速生成初始内容。

### images/ —— 图片资源

事件编辑器中可供选择的背景图、UI 图片、战斗单位图片等。

### template/ —— 角色模板

新建角色时使用的默认数据模板。当你创建角色但不选择英雄模板时，角色数据将以此为基础。

### md/ —— 使用说明

编辑器内置的操作指南文档，支持多语言。

## 路径查找顺序

编辑器按以下顺序查找 options 目录：

1. **用户设置** —— 你在设置中手动指定的路径（推荐）
2. **程序同级目录** —— 编辑器可执行文件旁边的 `options/` 文件夹

## 编辑器如何使用这些数据

| 你在编辑器中看到的 | 来源 |
|---|---|
| 新建角色时的英雄模板列表 | generated/ |
| 战斗配置中的攻击方式、技能选择 | generated/ |
| 物品掉落配置中的物品列表和图标 | generated/ |
| 事件编辑器中的背景图、立绘选择 | images/ |
| 事件编辑器中的 BGM 和音效选择 | audio/ |
| 事件的初始模板内容 | event/ |
| 各种选项的多语言显示 | generated/ |

## 设计说明

**为什么把基础数据独立出来？**

将基础数据与编辑器分离，好处是：

- **独立更新** —— 更新 options 包即可，编辑器本身不需要更新
- **体积可控** —— options 包含大量图标和音频资源，独立存放避免编辑器体积过大
