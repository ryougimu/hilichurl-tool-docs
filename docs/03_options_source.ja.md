# 第3章：基礎データ（Options）

> **注意**：Options のディレクトリ構造は開発中です。今後のバージョンで変更される可能性があります。内部の詳細を深く理解する必要はなく、役割と設定方法だけ把握すれば十分です。

## Options とは

エディタ自体はツールに過ぎません。ゲーム内にどのヒーローテンプレート、アイテム、効果音、背景などが存在するかを知る必要があります。それによって編集画面で適切な選択肢を提供できます。

これらの基礎情報は **options** というディレクトリに集約されています。エディタの「素材ライブラリ」と考えてください。エディタはここからデータを読み込み、ドロップダウンリストを埋めたり、アイコンを読み込んだり、効果音のプレビューを再生したりします。

## 入手方法

options パッケージをダウンロードし、ローカルの任意の場所に解凍してから、エディタの**設定**（`menu.settings`）でそのディレクトリパスを指定します。ダウンロード先は[付録 - ダウンロード](08_appendix.md#downloads)を参照してください。

## ディレクトリ構造

options ディレクトリには次のような領域があります。

```
options/
├── generated/       ヒーロー、アイテム、スキルなどのゲーム基礎データ
├── audio/           効果音と BGM リソース
├── images/          背景などの画像リソース
├── event/           イベントテンプレートファイル
├── template/        新規キャラクター作成時のデフォルトテンプレート
└── md/              エディタ内蔵の利用ガイド（多言語対応）
```

### generated/ — ゲーム基礎データ

最も中核的な部分です。ヒーローテンプレート、アイテム、スキル、ミニオンユニットなどのゲーム基礎データとそのアイコンリソースが含まれます。キャラクター作成時に選ぶヒーローテンプレート、戦闘設定の各種選択肢、アイテムドロップリストなど、すべてここから来ています。

### audio/ — 効果音と BGM

イベントエディタで選択可能な効果音と BGM リソースです。

### event/ — イベントテンプレート

事前定義されたイベントテンプレートファイルです。各ファイルは1種類のイベントタイプに対応します（例：`captured.json`、`give_birth_a.json` など）。新規イベント作成時、これらのテンプレートから初期内容を素早く生成できます。

### images/ — 画像リソース

イベントエディタで選択可能な背景、UI 画像、戦闘ユニット画像などです。

### template/ — キャラクターテンプレート

新規キャラクター作成時に使用されるデフォルトデータテンプレートです。ヒーローテンプレートを選ばずにキャラクターを作成する場合、このベースラインから始まります。

### md/ — 利用ガイド

エディタ内蔵の操作ガイド（多言語対応）。

## パス検索順序

エディタは次の順序で options ディレクトリを探します。

1. **ユーザー設定** — **設定**で手動で指定したパス（推奨）  
2. **実行ファイルと同じ階層** — エディタ実行ファイルの隣にある `options/` フォルダ

## エディタでの利用方法

| エディタ内で見える内容 | データ元 |
|----------------------|----------|
| キャラクター作成時のヒーローテンプレート一覧 | generated/ |
| 戦闘設定の攻撃方式・スキル選択 | generated/ |
| アイテムドロップ設定のアイテムリストとアイコン | generated/ |
| イベントエディタの背景・立ち絵選択 | images/ |
| イベントエディタの BGM と効果音選択 | audio/ |
| イベントの初期テンプレート内容 | event/ |
| 各種オプションの多言語表示 | generated/ |

## 設計上のメモ

**なぜ基礎データを独立させるのか**

基礎データをエディタから分離する利点：

- **独立更新** — options パッケージを更新するだけで済み、エディタ本体は更新不要  
- **サイズ制御** — options には大量のアイコンや音声ファイルが含まれます。独立配置することでエディタが肥大化しません
